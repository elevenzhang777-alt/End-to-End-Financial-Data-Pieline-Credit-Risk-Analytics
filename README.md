# End-to-End Financial Data Pipeline & Credit Risk Analytics

## Project Overview
This project demonstrates a professional financial data lifecycle: extracting raw transaction records via SQL, automating data cleansing using Python, and delivering actionable business insights through an interactive Power BI dashboard.

The primary goal is to replace inefficient manual Excel workflows with a scalable, automated pipeline that ensures Data Integrity and enhances Internal Audit efficiency.

## Tech Stack
- SQL: Raw data extraction and initial field filtering.
- Python (Pandas & NumPy): Automated data cleansing, deduplication, and risk flagging.
- Power BI**: Advanced data visualization and regional performance analysis.

## Key Features & Business Logic
1.  Automated ETL Pipeline: Cleanses messy financial datasets in seconds, handling whitespaces, inconsistent date formats, and duplicates.
2.  Data Integrity Handling: Automatically identifies and manages missing values (Nulls) to ensure the stability of downstream reporting.
3.  Smart Risk Flagging: Implements a business rule to automatically categorize invoices > $2,000 as "High Risk", allowing the audit team to prioritize high-value transactions.
4.  Regional Insights: Interactive dashboard enables managers to filter sales performance and risk distribution by state (QLD, NSW, VIC, etc.).

## Repository Structure
- `extract_sales.sql`: SQL script for database querying.
- `data_cleaning_pipeline.py`: Python automation script.
- `raw_sales_data.csv`: Sample of "messy" raw financial data used for testing.
- `cleaned_audit_report.csv`: The final, audit-ready dataset generated by Python.
- `Financial_Risk_Analysis.pbix`: Power BI source file containing the data model.
- `dashboard_preview.png`: Screenshot of the final analytical dashboard.

## Impact
By implementing this pipeline, the time required for monthly data reconciliation and risk screening is reduced from hours to seconds, significantly minimizing the risk of human error in financial reporting.
